设计

由于是想要设计成RESTful，所以自然地就想到了这样的结构：

服务器收到一个请求，根据这个请求的路径，分发器Dispatcher分发到对应的Handler。在Handler中根据请求的方法（GET/POST/DELETE等）执行相应的逻辑。

既然一个请求的处理流程现在已经清楚了，那么现在回到我们的最初的设计，Dispatcher放到哪里？

Dispatcher需要做的是解析请求的头部（获取路径嘛），然后分发给对应的Handler，这里有两种做法，继承Server/继承Handler。我选择后一种，继承BaseHTTPRequestHandler，为什么，因为里面有个parse_request可以用来解析请求头部。

总结

其实上面写了很多分析的过程，记录下来主要是积累经验。侯捷不是说过：源码之前，了无秘密。看源码也是个人能力的一种体现，需要多多锻炼才是啊。

虽然上面的代码很基础，但是还是有一些收获的。

首先，必须要带着目的去看，不要为了看源码而看源码。比如说上面，我想弄出Dispatcher和Handler的这种设计出来，那么就顺着request去看。

其次，有些问题看似简单，但是深挖下去也是能学到很多知识的。比如说，Handler与路径解耦这件事，如何做到路径改变不需要改变Handler内部的逻辑。
